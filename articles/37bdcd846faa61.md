AIエージェントにコーディングを依頼する際、開発者テストにおける「受け入れ基準」をAIに明確に伝えることが重要になります。しかし、期待するリクエスト・レスポンスを自然言語で書くと曖昧さが残りやすく、AIに寄り添って `curl` で記載しようとすると手間がかかるという課題がありました。

<!-- textlint-disable ja-technical-writing/no-doubled-joshi -->
そんな理由で人間にもAIにもやさしいツールを探してみたところ、プレーンテキストの力をフルに活用できる[Hurl](https://hurl.dev/docs/hurl-file.html)というHTTPクライアントが便利そうだったので紹介します。
<!-- textlint-enable ja-technical-writing/no-doubled-joshi -->

## Hurlとは

HurlはプレーンテキストでHTTPリクエストを定義し、実行できるCLIツールです。使い方は簡単で、独自のフォーマットである `.hurl` ファイルにHTTPリクエストとアサーションを書いて、`hurl` コマンドを実行するだけです。Rustで実装されており、内部的には[libcurl](https://curl.se/libcurl/)を呼び出すcurl互換の作りになっています。

```hurl:req.hurl
GET https://example.com/todos

[Query]
offset=100
limit=2
```

```shell
$ hurl ./req.hurl
```

## 実践的な使い方

### テンプレート記法

Hurlファイルでは `{{variable}}` のようなテンプレート記法を使うことができます。これにより、リクエスト時に決まるパラメータをコマンドライン引数として渡すことができます。

```hurl:req.hurl
GET https://example.com/todos
[Query]
offset={{offset}}
limit={{limit}}
```

```shell
$ hurl ./req.hurl --variable offset=100 limit=2
```

### 認証と環境変数

認証トークンなどのシークレットはテキストにベタ書きするのではなく、環境変数を使って定義したいです。Hurlでは `HURL_xxx` という特別な名前の環境変数をエクスポートすることで、Hurlファイルから読み込むことができます。

```hurl:req.hurl
GET https://example.com/todos
Authorization: Bearer {{token}}
```

```shell
$ export HURL_token=xxxxxxxxxxxxxxxxx

$ hurl ./req.hurl
```

### アサーション (検証)

HurlはHTTPクライアントとしての用途だけでなく、本記事で紹介するようなAPIテストを行うためのアサーション機能も備えています。例えば以下のHurlファイルでは、期待するHTTPステータスコードとレスポンスボディを記載しています。

```hurl:req.hurl
GET https://example.com/todos
HTTP 200
{
  "todos": [
    {
      "id": 1,
      "title": "todo 1"
    },
    {
      "id": 2,
      "title": "todo 2"
    }
  ],
  "count": 300,
  "offset": 102
}
```

APIが `500 Internal Server Error` を返したり、レスポンスデータが期待値に沿わない場合など、`hurl` コマンドは `0` 以外の終了コードを返して失敗します。

```shell
$ hurl req.hurl --test
error: Assert status code
  --> req.hurl:17:2
   |
   | GET https://example.com/todos
   | ...
16 | HTTP 200
   |      ^^^ actual value is <500>
   |
```

https://hurl.dev/docs/manual.html#exit-codes

## プロンプトの受け入れ基準で使う

最後にAIエージェントに渡すプロンプトの受け入れ基準で、Hurlのアサーション機能を用いた例を紹介します。以下のAPIはユーザーを作成するもので、成功するとデータなしの `201 Created` レスポンスを返します。

```hurl:tmp/post_shop_users.hurl
POST http://localhost:4001/protected/shop-users
HTTP 201

# リクエストヘッダー
Content-Type: application/json
Authorization: Bearer {{token}}

# リクエストボディ
{
  "shopId": "C1",
  "username": "yamamura_{{shop_user_id}}",
  "email": "yamamura+{{shop_user_id}}@wd-flat.com",
  "role": "manager"
}

[Asserts]
body isEmpty
```

意図的に不具合を混入したうえで、次のようなプロンプトを与えてClaude Codeに設計と実装をさせました。

## おわりに

今回はプレーンテキストでAPIテストができるHurlを紹介しました。シンプルな構文で書けるため学習コストが低く、ドキュメントをMarkdownでダウンロードできることから、開発者目線でもAI目線でもやさしい印象です。このような、**AIが実行できる受け入れ基準を書くスキル**は今後より重要になっていくと思いました。
